## 02. De dónde surge

### Título

¿Por qué este programa?

### Texto

La adopción de IA en tecnología ya no va solo de probar herramientas. Implica entender cómo incorporarlas en procesos reales: ideación, documentación, desarrollo, revisión, seguridad, testing y colaboración entre equipos.

Este programa surge para acompañar ese cambio de forma estructurada: con una herramienta base, un marco común de buenas prácticas y sesiones diseñadas para que los equipos puedan aplicar IA generativa sobre casos de uso reales.

### Bloque destacado

No se trata de usar Claude como un chat aislado.  
Se trata de aprender a integrarlo como una capa de trabajo: contextual, técnica, segura y medible.

---

## 03. Objetivo del programa

### Título

Objetivo

### Texto principal

Desarrollar capacidades reales de IA generativa en toda la organización de tecnología, integrando nuevos skills en el trabajo diario de las personas y activando casos de uso con impacto accionable y medible.

### Tres ideas clave

**Aprender fundamentos y límites reales de la IA generativa.**

**Aplicar Claude en flujos técnicos concretos:** specs, código, QA, documentación y agentes.

**Cerrar el programa con casos de uso demostrables, métricas y aprendizajes compartidos.**

## 04. De usar IA a trabajar con IA

### Título

De usar IA a trabajar con IA

### Intro

La edición Claude está pensada para que los equipos de tecnología pasen de interacciones sueltas con IA a flujos más consistentes, reutilizables y seguros.

### Bloques

**Antes**  
Uso puntual de prompts, respuestas aisladas y poca trazabilidad.

**Durante**  
Prácticas guiadas, contexto compartido, Claude Code, skills, MCPs y revisión de buenas prácticas.

**Después**  
Flujos reutilizables, subagentes, revisión previa de PRs, documentación y métricas de impacto.

### Frase de cierre

La adopción no ocurre cuando una herramienta está disponible. Ocurre cuando cambia la forma de trabajar.

## 05. Plan por semanas

### Título

Plan operativo de activación

### Intro

El programa combina sesiones presenciales, trabajo por grupos, autoconsumo, práctica inmersiva y espacios plenarios para consolidar aprendizajes.

### Semana 01 · Visión estratégica IA y configuración

Masterclass inspiracional, visión estratégica, impacto en educación y negocio, Claude como herramienta elegida y plan de despliegue.

**Duración:** 2 h  
**Formato:** presencial + plenario  
**Incluye:** caso inspiracional, visión estratégica, principales casos de uso y plan de despliegue.

### Semana 01 · Fundamentos de la IA

Tipos de IA, LLMs, tokens, costes, ventana de contexto, modelos Claude, guardarraíles y entorno tecnológico.

**Duración:** 3 h  
**Formato:** presencial, grupos por nivel  
**Práctica:** generación automática de specs y desarrollo con Claude Code.

### Semana 02 · Prompt engineering, skills y MCP

Método Contexto + Rol + Tarea + Output, prompting iterativo, skills, MCPs, connectors y Workbench.

**Duración:** 4 h  
**Formato:** presencial, grupos  
**Práctica:** uso de Claude Workbench para refinar prompts complejos.

### Semana 03 · Arquitectura, cumplimiento y seguridad

Arquitectura tecnológica, uso responsable de Claude, riesgos legales, auditoría de código y checklist de seguridad.

**Duración:** 1 h  
**Formato:** presencial, grupos  
**Práctica:** uso de Claude para buscar vulnerabilidades en fragmentos de código propios.

### Semana 04 · Agentes y subagentes

Agentes, subagentes, Claude Code, Agent SDK, sistemas multiagente y límites reales de cada camino.

**Duración:** 3 h  
**Formato:** presencial, grupos  
**Reto:** crear y documentar un subagente para Claude Code.

### Semana 05 · Testing, QA automatizado y flujos avanzados

Tests unitarios, integración, end-to-end, revisión de PRs con IA, CLI tools, sesiones múltiples y remote control.

**Duración:** 3 h  
**Formato:** presencial, grupos  
**Práctica:** implementar un AI Code Review System que pre-evalúe Pull Requests antes de la intervención humana.

### Semana 06 · Hack Day AI

Equipos de trabajo, producto mínimo viable, documentación con Claude Code, métricas de impacto y Demo Day.

**Duración:** 6 h  
**Formato:** presencial + plenaria  
**Práctica:** producto DEV completo generado y documentado con Claude Code.

## 06. Experiencia de aprendizaje

### Título

Una experiencia diseñada para practicar, no solo escuchar

### Texto

Cada bloque combina teoría aplicada, práctica inmersiva y retos conectados con el trabajo real del equipo. La intención es que cada sesión deje algo accionable: un prompt mejorado, una práctica validada, una configuración reutilizable, una revisión de seguridad o un caso de uso listo para evolucionar.

### Cards

**Teoría aplicada**  
Conceptos claros, aterrizados al contexto técnico y organizativo.

**Práctica inmersiva**  
Ejercicios sobre casos reales, no demos genéricas.

**Retos**  
Actividades para consolidar lo aprendido y documentar buenas prácticas.

## 07. Resultados esperados

### Título

Qué debería quedar al finalizar

### Texto

Al terminar el programa, el equipo no solo habrá recibido formación sobre IA generativa. Debería contar con una base común para usar Claude de forma más consistente, segura y conectada con procesos reales de tecnología.

- Un lenguaje común sobre IA generativa y sus límites.
- Buenas prácticas compartidas para prompting técnico.
- Primeros flujos de trabajo con Claude Code.
- Prompts, skills o configuraciones reutilizables.
- Checklist de seguridad aplicado a prácticas del programa.
- Casos de uso documentados.
- Métricas iniciales de impacto.
