# Componentes sugeridos

## Header

- Nombre corto del programa.
- Navegación por anclas.
- CTA pequeño: Ver plan.

## Hero

- Eyebrow.
- H1.
- Subheadline.
- CTAs.
- Datos rápidos.
- Panel visual tipo workspace.

El panel debe ser abstracto, no una captura real.

Módulos sugeridos:

- `context`
- `Claude.md`
- `skills`
- `QA review`
- `agent workflow`

## StatCard

Campos:

- Label.
- Value.
- Optional description.

## SectionIntro

Campos:

- Eyebrow opcional.
- Title.
- Description.
- Optional callout.

## CalloutBlock

Usar para destacar ideas centrales.

Estilo:

- Fondo gris claro o rojo extra light.
- Borde suave.
- Texto breve.
- Sin comillas decorativas grandes.

## BeforeDuringAfter

Columnas:

- Antes.
- Durante.
- Después.

## Timeline

Usar `weeks` desde `mockdata.ts`.

Cada item debe mostrar:

- Semana.
- Título.
- Duración.
- Formato.
- Descripción.
- Tags.
- Práctica o reto, si aplica.

En mobile: timeline vertical.

## LearningExperienceGrid

Cards:

- Teoría aplicada.
- Práctica inmersiva.
- Retos.
- Métricas.

## CapabilityGrid

Usar `capabilities` desde `mockdata.ts`.

## ResultsList

Mostrar qué queda al finalizar el programa.

Puede organizarse en dos columnas en desktop.

## Repository

Debe incluir:

- Filtros por tipo de recurso.
- Estado vacío.
- Cards de sesiones.
- Estados:
  - Próximamente.
  - Disponible.
  - En revisión.

## RepositoryCard

Campos:

- Week.
- Title.
- Status.
- Type.
- Description.
- Resources.
- Buttons.

Botones futuros:

- Ver video.
- Leer transcripción.
- Descargar recursos.
- Ver prompts.

## FinalCTA

Bloque de cierre con:

- Título.
- Texto breve.
- CTA principal.
- CTA secundario.
